- `01_nas_db.md` 완료 (DB 유저/권한)

## 1. api/.env 작성 (로컬 개발용)

```bash
cp api/.env.example api/.env

```

```dotenv
DB_PORT=5432
DB_NAME=budget_db
DB_USER=budget_book
DB_PASSWORD=실제_비밀번호

OLLAMA_URL=http://192.168.45.XXX:11434   # Mac Mini 고정 IP

```

---

## 2. K8s Namespace 생성

```bash
kubectl apply -f k8s/namespace.yaml
```

---

## 3. Secret 적용

`k8s/secret.yaml`에서 `CHANGE_ME`와 `MACMINI_IP` 값을 실제 값으로 교체 후:

```bash
kubectl apply -f k8s/secret.yaml
```

> secret.yaml은 더미값 상태로 git에 커밋. 실제 값은 로컬에서만 수정 후 apply.

---

## 4. 나머지 리소스 적용

GitHub Actions가 이미지를 빌드한 뒤 k8s/*.yaml 이미지 태그를 업데이트하면 ArgoCD가 자동 배포함.
최초 1회는 수동으로 apply 가능:

```bash
kubectl apply -f k8s/
```

---

## 5. ArgoCD 앱 등록

ArgoCD UI (`http://192.168.45.201`) 접속 → `+ New App`:

| 항목 | 값 |
|------|----|
| Application Name | `budget-book` |
| Repository URL | `https://github.com/<org>/budget-book` |
| Path | `k8s/` |
| Cluster | in-cluster |
| Namespace | `budget-book` |
| Sync Policy | Automatic |

---

## 6. 접속 확인

MetalLB가 `192.168.45.202`를 Ingress에 할당했는지 확인:

```bash
kubectl get svc -n budget-book
kubectl get ingress -n budget-book
```

브라우저에서 `http://192.168.45.202` 접속.
